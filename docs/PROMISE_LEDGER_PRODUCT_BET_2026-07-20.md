# Promise Ledger Product Bet

Date: 2026-07-20

## The problem worth proving

Teams make customer commitments in calls, meetings, and chat. The record can exist while responsibility does not. A generic memory system preserves the conversation; Promise Ledger preserves the obligation.

## The product truth

Useful operations memory is not remembering everything. It is keeping a promise visible until a named person owns the next move.

## First judge-visible slice

Fixture: a customer expects a Patio Permit answer by Wednesday.

1. The commitment is captured with its source.
2. No owner is assigned, so the screen opens on `OWNER REQUIRED`.
3. A user assigns Jordan Kim.
4. The assignment becomes an immutable event.
5. Only context matching the customer and project is eligible for a reviewable draft.
6. A draft is prepared, never sent.
7. The record makes its scope rule visible: unrelated customer notes are excluded.

## Why this fits CockroachDB x AWS

The contest asks for agentic memory that lasts. Promise Ledger makes the durable object a commitment record with source evidence, owner changes, deadlines, scoped retrieval, and an append-only event trail. This product frame gives CockroachDB a meaningful role rather than treating a database as a checkbox.

## Explicit non-claims in this build

- The interface currently uses public-safe fixture data.
- It does not yet claim live CockroachDB persistence, CockroachDB tool calls, AWS deployment, or autonomous sending.
- The first integration increment is a persistent promise/event schema, then two named CockroachDB tools, then one small AWS delivery surface.

## Design direction

This should feel like a calm case file, not a chatbot or a generic AI dashboard. The missing owner is the opening tension. Evidence, context scope, and the decision record do the explaining.

## Pass condition for this slice

A judge can understand in under one minute:

- why a lost promise is an operational problem;
- how the user names the accountable owner;
- why the suggested follow-up uses scoped rather than broad memory;
- that the product prepares reviewable work but does not send on its own.
